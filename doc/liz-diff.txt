1. Introduction                                        *liz-diff-introduction*
                                                                    *liz-diff*

`liz-diff.nvim` is a lightweight Neovim git diff plugin written in pure
Lua. Pop open a floating window, type any git reference, and browse the
changed files, then open each one in a native side-by-side vimdiff split.
No external diff tool, no heavy UI, no plugin dependencies.

Features ~
    - Centered floating window with an input prompt and a navigable
      changed-files list.
    - Any git reference: branch, commit hash, tag, range (`main..HEAD`),
      or empty for unstaged working-tree changes.
    - Side-by-side vimdiff per file status: added files show the working
      file on the left and an empty, `(new file)`-marked reference pane
      on the right, deleted files show a `[deleted]` placeholder on the
      left and the reference content on the right, renamed files are
      treated as modified, and — in the |:LizDiff| list flow — binary
      files notify instead of crashing.
    - Next / previous file navigation: after opening a file from the
      list, jump to the next / previous changed file with `]f` / `[f`
      (or |:LizDiffNext| / |:LizDiffPrev|) without reopening the picker;
      wraps around at both ends.
    - In-memory cache per keyword: reopening the panel restores the last
      reference's results instantly.
    - Explicit refresh: <CR> always re-fetches on submit, and `R`
      refreshes the results list in place without leaving the float.
    - Cursor position remembered per keyword across re-opens.
    - Show both diff panes' paths: |:LizDiffPaths| blinks each pane's
      absolute (or `<ref>:<path>`) path above it for ~2s.
    - Zero dependencies beyond Neovim 0.9+ and git.

    - Neovim >= 0.9.0
    - git (on `$PATH`)

==============================================================================
3. Installation                                        *liz-diff-installation*

Using lazy.nvim: >lua
    {
      'LizardLiang/liz-diff.nvim',
      cmd = 'LizDiff',
      opts = {},
    }
<
Using packer.nvim: >lua
    use {
      'LizardLiang/liz-diff.nvim',
      config = function()
        require('liz_diff').setup()
      end,
    }
<

4. Usage                                                      *liz-diff-usage*

                                                                    *:LizDiff*
:LizDiff

    Opens a centered floating window with an input prompt. Type a git
    reference and press <CR> to load the changed files. Leave the
    prompt empty to see unstaged working-tree changes.

    Pressing <CR> in the prompt always re-runs `git diff` for the typed
    reference, even if it was already fetched this session, keeping
    unstaged working-tree diffs current. Pressing <CR> on a file in the
    results list closes the float and opens a vertical vimdiff split
    with the working tree on the left and the reference version on the
    right — the file open and every git call for the selection are
    scoped to the repository root resolved when the list was fetched,
    independent of Neovim's cwd at selection time.

    The reference pane is always read via `git show`, regardless of the
    file's listed status (a stale/cached status is never trusted to skip
    the read). If that read fails for a reason other than the file being
    absent from the reference tree, a WARN-level notification names the
    file — a blank reference pane is never silent or unexplained.

    If Neovim's current working directory is not inside a git work tree,
    |:LizDiff| warns "liz-diff: not a git repository" and does nothing.

                                                                  *liz-diff-pr*
PR / MR review ~

    Inside the |:LizDiff| prompt, prefix the keyword with `#` or `!`
    followed by a number to target a pull request / merge request instead
    of a raw git ref: >

    #123     " GitHub PR 123 (or GitLab MR 123)
    !45      " same — either prefix works for either forge
<
    The plugin lists the PR's changed files (its `base...head` diff) in the
    usual float. Pressing <CR> on a file opens a READ-ONLY side-by-side
    diff with the PR head on the LEFT and base on the RIGHT — matching the
    working-left / reference-right rule used everywhere else. Both panes
    come from git, so nothing on disk is edited (a PR head is a branch you
    are reviewing, not your working tree).

    Requirements (optional, only for PR/MR keywords — core |:LizDiff| stays
    dependency-free):
    - GitHub: the `gh` CLI, authenticated.
    - GitLab: the `glab` CLI, authenticated.

    The forge is auto-detected from the `origin` remote host. Commits that
    are missing locally (common when reviewing others' PRs) are fetched on
    demand with a brief "fetching #123…" notification. A missing CLI, an
    undetectable provider, or an unresolvable number yields a clear error,
    never a crash.

    Note: host detection keys off the strings `github` / `gitlab` in the
    remote URL, so GitHub Enterprise / self-hosted GitLab on custom domains
    are not auto-detected yet.

                                                          *liz-diff-navigation*
                                              *:LizDiffNext* *:LizDiffPrev*
File navigation ~

    After opening a file from the |:LizDiff| list, move to the next or
    previous file in that same list without reopening the picker: >

    ]f   or  :LizDiffNext     " diff the next file in the list
    [f   or  :LizDiffPrev     " diff the previous file
<
    The `]f` / `[f` keymaps are set buffer-locally on both diff panes, so
    they work from whichever pane is focused. Navigation WRAPS AROUND:
    next on the last file goes to the first, previous on the first goes to
    the last. Works for both raw-ref and PR/MR lists.

    The commands (and `require('liz_diff').next()` /
    `require('liz_diff').prev()`) work from anywhere and notify
    "liz-diff: no active file list" if no |:LizDiff| list has been opened
    from yet (e.g. after only |:LizDiffFile|, which has no list).

    Navigating also updates the remembered cursor so reopening |:LizDiff|
    lands on the file currently under review. Configure the keys with
    `keymap.next_file` / `keymap.prev_file` (set either to `false` to
    disable that mapping and use the commands only), see
    |liz-diff-configuration|.

                                                                *:LizDiffFile*
:LizDiffFile [{ref}]

    Diffs the file in the current buffer against a git reference
    ({ref}, default `HEAD`) with zero prompts — the floating window is
    skipped entirely. Also available as
    `require('liz_diff').open_current({ref})`.

    Side order: the working buffer (current buffer, including unsaved
    edits) is placed on the LEFT, and the reference/commit version is
    placed on the RIGHT, with focus returned to the left window. This is
    the one shared layout rule across every liz-diff view — |:LizDiff|'s
    list flow matches it too (working/new side LEFT, reference RIGHT). >

    :LizDiffFile          " current file vs HEAD
    :LizDiffFile main     " current file vs the main branch
<
    If the file does not exist at the given reference (a new or
    untracked file), the right pane opens empty instead of erroring,
    and its buffer name carries a ` (new file)` marker so the blank
    pane clearly means "absent at that ref" rather than a real empty
    file.

    An unresolvable reference (e.g. a typo'd branch name) also yields
    an empty right pane with no error, but WITHOUT the ` (new file)`
    marker — this is intended: only genuine file-absence at a
    resolvable reference gets the marker.

    Guards:
    - No file in the current buffer (e.g. an empty or special buffer):
      notifies "liz-diff: no file in current buffer" and does nothing.
    - Current buffer is already a liz-diff reference buffer: notifies
      "liz-diff: cannot diff a liz-diff reference buffer" and does
      nothing.
    - The buffer's own directory is not inside a git work tree: warns
      "liz-diff: not a git repository" and does nothing. This check is
      scoped to the file's directory, not Neovim's process cwd, so it
      still works when Neovim was launched from outside any repo.

                                                          *liz-diff-compare*
                    *:LizDiffAdd* *:LizDiffCompare* *:LizDiffList* *:LizDiffClear*
Compare two arbitrary files ~

    Stage any two files and open them side-by-side, completely
    git-agnostic (no `require('liz_diff.git')`, no git calls at all):

    :LizDiffAdd
        Stages the CURRENT buffer's file into a compare list holding at
        most two files. Paths are stored absolute and normalized so a
        file staged from a differently-separated path is recognized as
        the same entry; on Windows, paths are also case-folded so a
        differently-cased path to the same file (case-insensitive NTFS)
        is recognized too, while Unix paths stay case-sensitive.

        Guards (same sequence as |:LizDiffFile|'s current-buffer checks):
        no file name, a `liz-diff://` reference buffer, or a non-empty
        `buftype` each notify "liz-diff: no file in current buffer" (or,
        for a `liz-diff://` buffer, a message naming that case) and leave
        the list unchanged.

        Staging the same file twice notifies that it is already staged
        and leaves the list unchanged. Staging a third file while the
        list already holds a pair opens a floating picker listing the
        two staged files (with LEFT/RIGHT slot) plus the incoming file:
        press `1` or `2` to replace that slot (notifying the result), or
        `q` / `<Esc>` to cancel and leave the pair untouched.

    :LizDiffCompare
        Opens the two staged files as real, editable file buffers in a
        two-pane vimdiff: the FIRST-staged file on the LEFT, the
        SECOND-staged file on the RIGHT, forced regardless of
        `'splitright'` — the same shared layout rule as |:LizDiffFile|.
        Each buffer's name is simply its own file path. Any prior
        liz-diff reference panes are torn down first, matching
        |:LizDiffFile|.

        Fewer than two files staged: notifies "liz-diff: need two files
        to compare, N staged" and does nothing. A staged file that no
        longer exists on disk: warns naming the missing file and aborts
        before either pane opens.

    :LizDiffList
        Opens a floating window listing the currently staged files with
        their slot and LEFT/RIGHT assignment, or an empty-state message
        when nothing is staged. Closes on `q` / `<Esc>`.

    :LizDiffClear
        Empties the compare list and notifies confirmation.

    `<Plug>(LizDiffAdd)` and `<Plug>(LizDiffCompare)` are non-recursive
    mappings for the same actions, for users who prefer to bind their
    own keys — no default key is bound: >lua

    vim.keymap.set('n', '<leader>ca', '<Plug>(LizDiffAdd)')
    vim.keymap.set('n', '<leader>cc', '<Plug>(LizDiffCompare)')
<
                                                              *:LizDiffPaths*
:LizDiffPaths

    Blinks the path of BOTH panes of the currently active diff as virtual
    text (extmark virt_lines, highlighted as |hl-Comment|) pinned above
    each pane, for about 2 seconds, then auto-clears. Real buffer content
    is never modified.

    - An on-disk pane (a real file buffer) shows its full absolute path
      (`fnamemodify(name, ':p')`).
    - A `liz-diff://` reference/scratch pane (a commit, PR blob, or the
      `[deleted] <path>` placeholder) shows `{ref}:{repo-absolute path}`,
      e.g. `HEAD:/repo/a.lua` — computed and stashed on the pane's buffer
      when it was created, so the command needs no extra git call and
      makes no assumption about Neovim's current working directory.
    - Running |:LizDiffPaths| again re-renders and restarts the 2-second
      timer. Opening a new diff (via any liz-diff command) clears any
      overlay left over from a previous |:LizDiffPaths| call.
    - With no diff window open in the current tabpage, it notifies
      "liz-diff: no active diff" and makes no other change.

    Also available as `require('liz_diff').paths()`.
<

5. Keymaps                                                  *liz-diff-keymaps*

These keymaps are active inside the |:LizDiff| floating window only (they
do not apply to |:LizDiffFile|, which has no float):

    Key            Action ~
    <CR>           Open the file under the cursor in vimdiff.
    R              Refresh the file list for the current reference in
                   place, preserving cursor position. No-op until a
                   reference has been submitted at least once.
    j / k          Navigate the results list.
    <Esc> / q      Close the float.
    i / /          From the results list, refocus the prompt for
                   insert mode.

These keymaps are active on the two diff panes after a file is opened
(see |liz-diff-navigation|):

    Key            Action ~
    ]f             Diff the next file in the list (wraps to the first).
    [f             Diff the previous file in the list (wraps to the last).

The close, open-diff, refresh, and next/prev-file keys are configurable,
see |liz-diff-configuration|.

6. Configuration                                      *liz-diff-configuration*

                                                              *liz-diff-setup*
require('liz_diff').setup({opts})

    Merges {opts} into the default configuration. All fields are
    optional; call with no arguments to use the defaults as-is. >lua

    require('liz_diff').setup({
      width  = 0.8,       -- float width as fraction of editor width
      height = 0.6,       -- float height as fraction of editor height
      border = 'rounded', -- 'rounded' | 'single' | 'double' | 'none'
      keymap = {
        close     = { '<Esc>', 'q' },
        open_diff = '<CR>',
        refresh   = 'R',
        next_file = ']f',
        prev_file = '[f',
      },
    })
<
    Fields ~
        width       (number) Float width as a fraction of `vim.o.columns`.
                    Default: `0.8`
        height      (number) Float height as a fraction of `vim.o.lines`.
                    Default: `0.6`
        border      (string) Float border style: 'rounded', 'single',
                    'double', or 'none'.
                    Default: `'rounded'`
        keymap.close      (string[]) Keys that close the float, bound in
                    both the prompt and results buffers.
                    Default: `{ '<Esc>', 'q' }`
        keymap.open_diff  (string) Key bound in the results buffer that
                    opens the file under the cursor in vimdiff.
                    Default: `'<CR>'`
        keymap.refresh    (string) Key bound in the results buffer that
                    re-runs the diff for the current reference in place.
                    Default: `'R'`
        keymap.next_file  (string) Key bound on the diff panes that opens
                    the next file in the list. `false` disables it.
                    Default: `']f'`
        keymap.prev_file  (string) Key bound on the diff panes that opens
                    the previous file in the list. `false` disables it.
                    Default: `'[f'`

7. API                                                          *liz-diff-api*

require('liz_diff').setup({opts})              See |liz-diff-setup| above.

require('liz_diff').open()
    Opens the |:LizDiff| floating window. Equivalent to running
    |:LizDiff|.

require('liz_diff').open_current({ref})
    Diffs the current buffer's file against {ref} (default `HEAD`).
    Equivalent to running |:LizDiffFile| {ref}. See |:LizDiffFile| for
    full behavior, including side order and guards.

require('liz_diff').next()
require('liz_diff').prev()
    Opens the next / previous file in the active |:LizDiff| list, wrapping
    at the ends. Equivalent to |:LizDiffNext| / |:LizDiffPrev|. No-op with
    a notification when no list is active. See |liz-diff-navigation|.

require('liz_diff').add()
    Stages the current buffer's file into the compare list. Equivalent to
    |:LizDiffAdd|. See |liz-diff-compare|.

require('liz_diff').compare()
    Opens the two staged compare-list files side-by-side. Equivalent to
    |:LizDiffCompare|. See |liz-diff-compare|.

require('liz_diff').list()
    Shows the staged compare-list files in a float. Equivalent to
    |:LizDiffList|. See |liz-diff-compare|.

require('liz_diff').clear()
    Empties the compare list. Equivalent to |:LizDiffClear|. See
    |liz-diff-compare|.

require('liz_diff').paths()
    Blinks the path of both panes of the active diff for ~2s. Equivalent
    to |:LizDiffPaths|.
